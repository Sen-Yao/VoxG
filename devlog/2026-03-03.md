## 🔧 修改内容

### 1. `utils.py` - 核心正交化逻辑

**修改函数：** `node_neighborhood_feature`

**新增功能：**
- 添加 `orthogonalize` 参数
- 实现 Gram-Schmidt 正交化：对多跳传播特征做正交化，减少信息冗余
- 正交化后归一化，保持特征尺度

```python
# 正交化核心逻辑
if orthogonalize and k > 1:
    for i in range(len(all_hops) - 1):
        # 计算投影并减去
        proj_coef = torch.sum(result * all_hops[i], dim=1, keepdim=True) / (torch.sum(all_hops[i] * all_hops[i], dim=1, keepdim=True) + 1e-8)
        result = result - proj_coef * all_hops[i]
    # 归一化保持尺度
    result = torch.nn.functional.normalize(result, p=2, dim=1) * torch.sqrt(torch.tensor(features.shape[1], dtype=features.dtype))
```

## ❓ 问题分析

### 正交化未生效的可能原因

1. **正交化位置不对**
   - 当前：在每跳传播后正交化
   - 可能更好的位置：在 concat 所有跳之后做整体正交化

2. **正交化强度过大**
   - Gram-Schmidt 可能过度消除了有用信息
   - 考虑软正交化（保留部分原始信息）

3. **参数配置问题**
   - `alpha` 残差系数可能已经起到了类似正交化的作用
   - 需要调整 `alpha` 和正交化的组合

4. **数据集特性**
   - Amazon 数据集的多跳特征本身可能已经相对独立
   - 需要在更复杂的数据集上测试（如 Reddit、DGraph）

## 💡 下一步优化方向

### 方案 1：软正交化（推荐）
```python
# 不完全正交化，保留部分原始信息
beta = 0.5  # 正交化强度
result = (1 - beta) * result + beta * orthogonalized_result
```

### 方案 2：注意力加权正交化
- 用注意力机制学习每跳的权重
- 对加权后的特征做正交化

### 方案 3：正交正则化损失
- 不在前向传播正交化
- 在损失函数中添加正交正则项
```python
# L_orth = sum((hop_i · hop_j)^2) for i != j
loss += lambda_orth * L_orth
```

### 方案 4：QR 分解
- 用 QR 分解代替 Gram-Schmidt
- 数值稳定性更好

## 🔍 正交化路线失败分析

### 失败原因

1. **多跳特征相关性是有效信息**
   - VecGAD 的 PPR 传播中，相邻跳特征高度相关（相关系数>0.8）
   - 这种相关性可能编码了重要的图结构信息
   - 强制正交化消除了这些有用信息

2. **Amazon 数据集特性**
   - 可能是同质图，多跳传播本身就提供平滑的特征
   - 正交化破坏了这种平滑性

3. **正交化方向错误**
   - Gram-Schmidt 是硬正交化，可能过于激进
   - 需要更柔和的去相关策略

### 经验教训

**核心原则：如果性能指标完全持平，代码一定有问题！**

- 正交正则化 lambda=0.01/0.1 时性能完全持平 → 代码有问题或 lambda 太小
- 调试后发现代码正确，但 lambda 需要更大（1.0+）
- 即使 lambda=1.0/10.0，性能依然持平 → 正交化对 Amazon 数据集无效
