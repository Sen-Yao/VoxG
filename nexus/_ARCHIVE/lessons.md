# Nexus Lessons Learned

## 2026-03-19: 架构创新系统性失败

### 核心教训
**VoxG 架构已高度优化，架构改进边际效应为负**

### 已验证失败的方向

1. **位置编码 (Cosine/Learnable PE)**
   - 结果: -14% ~ -32% AUC
   - 原因: PPR tokenization 已隐式编码位置信息
   - 教训: 显式位置编码是冗余甚至噪声

2. **稀疏注意力 (Exphormer)**
   - 结果: -31% AUC
   - 原因: 与 hop tokenization 不兼容
   - 教训: 注意力机制需要与 tokenization 协同设计

3. **对比学习 (CVGAD 风格)**
   - 结果: -27% ~ -29% AUC
   - 原因: 与 Ring Loss 目标冲突
   - 教训: 损失函数设计需要考虑与现有损失的交互

4. **Focal Loss**
   - 结果: +0.5% AUC
   - 原因: 类别不平衡不是主要问题
   - 教训: 需要深入理解数据集特性

### 关键洞察

1. **PPR tokenization 的优势**
   - 隐式编码位置信息
   - 隐式编码图结构信息
   - 避免引入额外噪声

2. **Ring Loss 的独特性**
   - 强制正常样本在超球面上形成紧凑表示
   - 与重构损失协同工作
   - 其他损失函数可能破坏这种平衡

3. **图异常检测的特殊性**
   - 不适合 CV/NLP 中常用的数据增强策略
   - 节点特征的结构性比像素/文本更强
   - 需要领域特定的创新方法

### 未来建议

1. **避免直接迁移 CV/NLP 技术**
   - 需要深入理解图异常检测的特性
   - 小规模验证后再大规模应用

2. **关注更温和的改进**
   - 学习率调度优化
   - 正则化策略
   - 集成学习

3. **深入理解现有架构优势**
   - 分析 VoxG 为何已如此优秀
   - 寻找真正的瓶颈
