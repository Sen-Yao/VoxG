## 一、重大错误：评估方法错误

### 错误描述

**所有实验使用了错误的评估方法**：无监督马氏距离

**正确方法**：有监督分类器（正常标签 + 伪异常 → BCE 训练）

### 影响范围

| 数据集 | 无监督 AUC | 有监督 AUC | 错误影响 |
|--------|-----------|-----------|----------|
| Photo | 0.68 | **0.99** | +44.6% |
| Amazon | 0.84 | **0.97** | +15.0% |
| Elliptic | 0.30 | **0.70** | +131.5% |

### 受影响的实验

| 实验 | 状态 | 需要重跑 |
|------|------|----------|
| Channel-wise Token | 全部无效 | ✅ |
| RHO 实验 | 部分无效 | ✅ |
| 所有马氏距离评估的实验 | 无效 | ✅ |

## 二、错误根因分析

### 根本原因

1. **没有仔细阅读原 VoxGFormer 代码**
   - 原 VoxGFormer 使用有监督分类器
   - 我错误假设半监督 = 无监督评估

2. **半监督 GAD 的正确理解**
   ```
   半监督 = 正常节点标签可获取 + 可生成伪异常
         → 有监督训练（BCE）
         → 不是无监督评估
   ```

3. **没有验证评估方法的正确性**
   - 应该先用简单 baseline 验证评估方法

## 三、Channel-wise Token 方案分析

### 方案是否有效？

**结论：可能无效，即使使用正确评估方法**

### 理由

| 问题 | 说明 |
|------|------|
| **信息冗余** | Prompt 已经可以学习任意重要性，强行加 Channel-wise 是冗余 |
| **Token 数量不稳定** | Photo 746 tokens vs Amazon 26 tokens，差异 28 倍 |
| **计算效率低** | 高维特征时 Token 过多 |
| **没有理论优势** | 不比原始 Prompt 方式更好 |

### 正确方向

| 优先级 | 方向 |
|--------|------|
| 🔴 高 | 修复评估方法，重跑原始 VoxGFormer |
| 🟡 中 | 改进伪异常生成策略 |
| 🟢 低 | Channel-wise Token（可能不需要） |

## 五、正确的工作流程

### 修复后的工作流

```
1. 输入：多跳传播 Token [N, K+1, D]
    ↓
2. Prompt 提取：[N, H]
    ↓
3. 伪异常生成：
   - 温度调整（原版）
   - 通道 mask（改进）
   - 噪声添加（改进）
    ↓
4. 有监督训练：
   - 正样本：伪异常
   - 负样本：正常节点
   - 损失：BCE
    ↓
5. 评估：AUC-ROC
```
